# 課題7 — 技術成熟度レベルTRLの推移

提出物7「技術成熟度レベルTRLの推移」用の章本体フォルダ．
親文書 `../reports.tex` の第4章として `\input` される．

## 構成

```
trl_history/
├── trl_history.tex   # 章本体（親文書から \input される）
└── README.md
```

## TRLとは

Technology Readiness Level — NASA 由来の技術成熟度評価指標．
1（基礎原理確認）から9（実運用実証済み）の9段階．
本プロジェクトでは試作開発の進行に応じて TRL 4〜6 を行き来する想定．

## 評価の単位

**FBS（機能分解構造）の各機能要素ごとに** TRLを評価する．
FBSは下記の5フェーズ・14機能で構成される：

| FBSフェーズ | 機能 |
|---|---|
| 指揮動作検出 | 拍タイミング検出 / テンポ推定 / フェルマータ検出 / 小節同期見逃し補完 |
| 指揮者人形動作 | 腕往復動作 / 輝度エンコードLED制御 |
| 演奏制御 | 輪唱オフセット制御 / 楽譜進行制御 / テンポ追従演奏 |
| 音響合成 | 音色合成（倍音・ADSR） / エフェクト付与 / フェルマータ延奏 |
| 状態管理 | 演奏状態遷移管理 / 異常時フォールバック |

## 追記の書式

`trl_history.tex` の各機能行を編集する．書式：

```
FBSフェーズ & 機能 & TRLレベル & 根拠 \\
```

フェーズ列は，各フェーズの最初の行にだけ書き，続く行は空欄にしてグループ化を示す．
グループの区切りには `\midrule` を挟む．

例：

```
指揮動作検出 & 拍タイミング検出 & TRL 4 & フォトTrによる拍検出が試験環境で動作確認済み \\
 & テンポ推定 & TRL 3 & EMAアルゴリズムのシミュレーション実証段階 \\
```

## ビルド方法

このフォルダ単独ではビルドしない．リポジトリルートで親文書をビルドする：

```bash
lualatex reports.tex
```

## 提出毎の更新

提出のたびに，各機能の現時点のTRLレベルと根拠を最新の値に書き換える．
過去の値は本表からは消えるが，`management_plan.tex` の変更履歴章（および課題3の独立変更履歴文書）で
更新事実を追えるようにしておく．
