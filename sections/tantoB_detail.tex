\section{担当B — センサ処理・状態機械}

担当 B は，子機 Arduino における指揮者人形の観測・テンポ推定・状態管理を担う．
センサ構成は赤外線センサ（距離取得）とフォトトランジスタ（明るさ検出）の 2 種類であり，
両センサの情報を組み合わせてテンポ状態を判定し，発音タイミングを決定する．

\subsection{ハードウェア設計}

赤外線センサ（GP2Y0A21YK）はアナログピン \texttt{A0} に接続し，
フォトトランジスタ（NJL7502L）はアナログピン \texttt{A1} に接続する．
電源電圧は $V_{\text{ref}} = 5.0\,\text{V}$ とする．

赤外線センサの出力電圧$V$から距離$d$ [cm] への変換は以下の近似式を用いる．

\begin{equation}
  d = 27.728 \times V^{-1.2045}
\end{equation}

フォトトランジスタは演奏開始前に \texttt{analogRead}（ピン \texttt{A1}）で輝度レベルを読み取り，自機IDに対応するレベルを検出した時点でスタートフラグを立てる．

輝度レベルと楽器IDの対応を以下に示す（親機PWMデューティ比と子機 \texttt{analogRead} 値の目安）．

\begin{lstlisting}[caption=担当 B 主要変数]
// 赤外線センサ
#define IR_PIN A0
volatile float irDistance = 0.0f;        // 赤外線センサによる距離 [cm]
const int IR_WINDOW_SIZE = 3;            // 移動平均のためのサンプル数
float irDistanceBuffer[IR_WINDOW_SIZE] = {0.0f}; // 距離バッファ
float calibratedIrValue = 0.0f;          // 正規化済み距離 [%]
float previousCalibratedIrValue = 0.0f;  // 前回の正規化済み距離 [%]

// ピーク検出用
const int IR_PEAK_WINDOW_SIZE = 3;
float calibratedIrHistory[IR_PEAK_WINDOW_SIZE] = {0.0f};
const float IR_PEAK_THRESH = 60.0f;     // ピーク検出閾値 [%]
const float IR_PEAK_MARGIN = 3.0f;      // ピーク検出マージン [%]

// フォトトランジスタ
#define PHOTO_PIN A1
int photoValue = 0;                      // フォトトランジスタ生値
float calibratedPhotoValue = 0.0f;       // 正規化済み受光量 [%]
const float BEAT_DETECT_THRESH = 50.0f;  // 拍検出閾値 [%]
const float PERFORM_START_THRESH = 85.0f; // 演奏開始閾値 [%]
bool beatDetected = false;               // 拍検出フラグ
bool photoHigh = false;                  // 拍検出中フラグ（チャタリング防止）

// キャリブレーション
unsigned long startCalibrate = 0;        // キャリブレーション開始時刻 [ms]
const unsigned long CALIBRATE_THRESH = 5000; // キャリブレーション所要時間 [ms]
float irMax = 0.0f;                      // 距離最大値 [cm]
float irMin = 9999.0f;                   // 距離最小値 [cm]
int photoMax = 0;                        // 受光量最大値
int photoMin = 9999;                     // 受光量最小値
bool isCalibrated = false;               // キャリブレーション済みフラグ

// EMA テンポ推定
float estimatedBeatInterval = 1000.0f;   // 推定拍間隔 [ms]
float emaAlpha = 0.2f;                   // EMA 係数
unsigned long currentBeatTime = 0;       // 現在の拍検出時刻 [ms]
unsigned long previousBeatTime = 0;      // 前回の拍検出時刻 [ms]
unsigned long nextBeatTime = 0;          // 次回の推定拍検出時刻 [ms]（担当Cと共有）
unsigned long predictedIrBeatTime = 0;   // 赤外線センサによる次拍予測時刻 [ms]
const float TEMPO_CHANGE_THRESH = 10.0f; // テンポ変化判定閾値 [%]
bool isTempoChanged = false;             // テンポ変化判定フラグ
bool beatUpdated = false;                // 拍更新フラグ（担当Cと共有）

// 状態機械
enum State {IDLE, CALIBRATE, PREBEAT, PLAYING, TEMPO_CHANGE, FERMATA};
State state = IDLE;                      // 現在の状態（担当Cと共有）
int prebeatCount = 0;                    // 予備振りカウント
const int PREBEAT_THRESH = 4;            // 必要な予備振り回数
int stableCount = 0;                     // 安定拍連続カウント
const int STABLE_THRESH = 3;             // 通常演奏復帰に必要な安定拍数
const float STOP_THRESH = 5.0f;          // 静止判定閾値 [%]（要調整）
int staticCount = 0;                     // 静止サンプル連続カウント
const int FERMATA_THRESH = 3;            // フェルマータ判定に必要な静止サンプル連続数
unsigned long lastBeatTime = 0;          // 最後に拍を検出した時刻 [ms]
const int MISSED_BEAT_THRESH = 3;        // 演奏中止に必要な未検出拍数
bool isPlaying = false;                  // 演奏中フラグ（担当Cと共有）

// その他
const float V_REF = 5.0f;               // 電源電圧 [V]
const int INSTRUMENT_ID = 0;            // 楽器ID
\end{lstlisting}

\subsubsection{担当Cとの共有変数}

担当 B と担当 C は同一の Arduino 上で動作するため，以下のグローバル変数を共有することで連携する．
担当 B が書き込み，担当 C が読み取る変数を以下に示す．

\begin{table}[H]
\centering
\caption{担当 C との共有変数一覧}
\begin{tabular}{p{3.5cm}p{2.5cm}p{4cm}p{4cm}}
\hline
\textbf{変数名} & \textbf{型} & \textbf{担当Bの役割} & \textbf{担当Cの使い方} \\
\hline
\hline
\texttt{nextBeatTime} & \texttt{unsigned long} &
次の拍の予測絶対時刻を書き込む & 信号送信タイミングの基準として読む \\
\hline
\texttt{beatUpdated} & \texttt{bool} &
\texttt{nextBeatTime} が更新されたら \texttt{true} にする &
\texttt{true} を検知して処理後 \texttt{false} に戻す \\
\hline
\texttt{state} & \texttt{State} &
現在の演奏状態を書き込む &
FERMATA 判定などに使う \\
\hline
\texttt{isPlaying} & \texttt{bool} &
演奏開始合図として \texttt{true} にする &
演奏開始のトリガーとして使う \\
\hline
\end{tabular}
\end{table}

なお，\texttt{beatUpdated} を \texttt{false} にリセットする責任は担当 C 側にある．
