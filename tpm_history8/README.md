# 課題8 — 技術性能指標TPMの推移

提出物8「技術性能指標TPMの推移」用の章本体フォルダ．
親文書 `../reports.tex` の第5章として `\input` される．

## 構成

```
tpm_history/
├── tpm_history.tex   # 章本体（親文書から \input される）
├── figures/          # TPM推移グラフ等の画像を置く（必要に応じて作成）
└── README.md
```

## TPMの定義

`management_plan.tex` の「TPM一覧」表に定義済み．現在の5項目：

| No. | 内容 | 目標 |
|---|---|---|
| TPM-1 | 距離計算誤差 | ≤ ±3 cm |
| TPM-2 | 拍検出成功率 | ≥ 95\% |
| TPM-3 | Arduino→Processing転送遅延 | ≤ 50 ms |
| TPM-4 | EMA（α=0.2）テンポ収束拍数 | ≤ 13 拍 |
| TPM-5 | 追従可能最大BPM | ≥ 104 bpm |

## 追記の書式

`tpm_history.tex` の表に，次の書式で行を追記する．

```
YYYY-MM-DD & TPM-X & 目標値 & 実測値 & ○/× & 備考 \\
```

例：

```
2026-05-29 & TPM-2 & $\geq 95\,\%$ & 92.4\% & × & 80 bpm試験，照明環境改善で再試験予定 \\
2026-06-05 & TPM-2 & $\geq 95\,\%$ & 96.1\% & ○ & 照明改善後の再試験で達成 \\
```

## ビルド方法

このフォルダ単独ではビルドしない．リポジトリルートで親文書をビルドする：

```bash
lualatex reports.tex
```

## 提出毎の更新

本ファイルは **累積** で運用する（過去の測定結果も残す）．
新しい測定があるたびに行を追加していく．
推移を折れ線グラフで可視化する場合は `tpm_history/figures/tpm_chart.pdf` 等を配置．
画像パスは親文書（`reports.tex`）から見た相対パスである点に注意．
